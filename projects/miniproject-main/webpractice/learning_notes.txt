covered in the HTML, CSS, and JS components.

-------------------------------------------------------
1. HTML (HYPERTEXT MARKUP LANGUAGE): The Structure
-------------------------------------------------------
• Semantic Tags:
  HTML5 tags like <header>, <nav>, <main>, <section>, 
  <article>, <aside>, and <footer> give meaning to your 
  layout. They improve Accessibility (screen readers) 
  and SEO (Search Engine Optimization).

• Typography & Formatting:
  - <h1> to <h6>: Headings (Only use one <h1> per page).
  - <p>: Standard paragraph text.
  - <strong>: Indicates important text (renders bold).
  - <em>: Indicates emphasized text (renders italicized).
  - <blockquote>: Used for quoting text from another source.

• Lists:
  - <ul>: Unordered List (bullet points).
  - <ol>: Ordered List (numbered).
  - <li>: List Item (used inside ul/ol).

• Images & Media:
  - <img>: The image tag. The `alt` attribute is crucial 
    for accessibility and acts as a fallback if the image 
    fails to load.
  - <figure> & <figcaption>: Bundles an image semantically 
    with a caption.

• Tables:
  - Should only be used for tabular data (not for page layout).
  - Constructs: <table>, <thead>, <tbody>, <tr> (row), 
    <th> (header cell), <td> (data cell).

• Forms & Inputs:
  - <form>: Collects user input. The `action` attribute specifies 
    where the data is sent.
  - <input>: Various types (text, email, checkbox, radio).
  - <label>: Should always be linked to inputs using the `for` 
    attribute (matching the input's `id`).
  - <select>: Creates a drop-down list.

-------------------------------------------------------
2. CSS (CASCADING STYLE SHEETS): The Presentation
-------------------------------------------------------
• CSS Variables (Custom Properties):
  Defined in the `:root` pseudo-class (e.g., `--primary-color: blue;`) 
  so they can be reused consistently across the stylesheet.

• Box Sizing & Universal Selector (*):
  Using `box-sizing: border-box` on all elements ensures that 
  padding and borders are included within an element's specified 
  width and height.

• The Box Model:
  Every HTML element is a rectangular box consisting of:
  - Margin: Space completely outside the element.
  - Border: The outline/border of the element.
  - Padding: Space inside the element, between content and border.
  - Content: The actual text or core media.

• Layouts (Flexbox & Grid):
  - Flexbox (`display: flex`): Excellent for 1-dimensional 
    layouts (aligning items in a single row or a single column).
  - CSS Grid (`display: grid`): Robust system for creating complex, 
    2-dimensional layouts (rows AND columns simultaneously).

• Pseudo-Classes:
  Used to style the "state" of an element.
  - `:hover` applies styles when the mouse is over an element.
  - `:focus` applies styles when an element (like an input) is active.
  - `:nth-child(even)` applies styles to alternating elements 
    (great for striped table rows).

• Media Queries (Responsive Design):
  Uses `@media screen and (max-width: 768px) { ... }` to 
  apply specific CSS rules only on smaller screens (like mobile devices).

-------------------------------------------------------
3. JAVASCRIPT: The Behavior & Interactivity
-------------------------------------------------------
• Variables:
  - `const`: For values that NEVER change (constants). Default to this.
  - `let`: For values that you expect to change/reassign later.
  - `var`: The old way of declaring variables (avoid using in modern JS).

• Data Types:
  - String ("Hello World"), Number (42.5), Boolean (true/false).
  - Array (Lists): `["HTML", "CSS"]`
  - Object (Key-Value pairs): `{ skill: "JS", level: "Beginner" }`

• DOM Manipulation (Document Object Model):
  JavaScript can select elements from the HTML to read or change them.
  - `document.getElementById('id-name')`
  - `document.querySelector('.class-name')`

• Functions:
  Reusable code blocks.
  - Standard Function: `function myFunc(param) { ... }`
  - Arrow Function: `const myFunc = (param) => { ... }` (Modern, compact syntax)

• Event Listeners:
  JavaScript "listens" for user interactions.
  - `button.addEventListener('click', () => { ... })`
  - On forms, listening to the 'submit' event and calling 
    `event.preventDefault()` is crucial to stop the page from reloading.

• Asynchronous JavaScript:
  Code that doesn't run immediately.
  - `setTimeout(() => { ... }, 2000)` executes code after a 2-second delay.
  - The `fetch()` API (not extensively covered, but essential) is used 
    to asynchronously pull data from web servers.
